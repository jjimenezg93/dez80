### Cyan Bullet movement explanation

Moving a cyan bullet means rotating through the 4 low order bits of each Byte, since the 4 high order bits must always be 0. Instead of convoluted jumping logic, this does the trick:

(HL)    - 0000 0011
(HL+1)  - 0000 0000
A       - 0000 0000

RRD
A       <- 0000 0011
(HL)    <- 0000 0000

SCF
CY      <- 1

CCF
CY      <- 0 ; !CY

RRA
A       <- 0000 0001
CY      <- 1

RLD
A       <- 0000 0000
(HL)    <- 0000 0001

RRA
A       <- 1000 0000

RRA
A       <- 0100 0000

RRA
A       <- 0010 0000

RRA
A       <- 0001 0000

RRA
A       <- 0000 1000

INC L
HL      <- HL+1

LD (HL), A
(HL+1)  <- 0000 1000
